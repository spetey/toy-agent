fb2d Language Reference

A 2D reversible programming language Reversible Turing-Complete

Execution Model

The instruction pointer (IP) moves on a toroidal grid. Each step:

  1. Read opcode at grid[IP]
  2. Execute instruction (may change IP direction)
  3. Advance IP one cell in current direction

Von Neumann architecture: code and data share the same grid. Any byte sequence is a valid program (undefined opcodes are NOPs).

Machine State

IP Instruction pointer (row, col)
DIR IP direction: N / E / S / W
CL Control locus (flat index) — branching
H0 Data head 0 (flat index) — primary data
H1 Data head 1 (flat index) — secondary data
GP Garbage pointer (flat index) — breadcrumbs

All head registers are flat indices: flat = row × cols + col. Grid cells are 8-bit (0–255, mod 256).

Mirror Reflection Rules

/
E↔N S↔W
\
E↔S N↔W

Conditional mirrors check grid[CL]: if condition is true, reflect; otherwise pass through (NOP).

NOP (byte 0, bytes 33–255)

Byte 0 (displayed as ·) and bytes 33–255 are NOPs — IP passes through with no effect. This makes the language "valid everywhere".

Reversibility

Every instruction has a well-defined inverse. The simulator reverses execution without storing history — it deduces the prior state from the current state alone.

At conditional mirrors, the condition is re-evaluated on reverse. Since the condition (grid[CL]) is not modified by the mirror, it correctly disambiguates the forward path.

Common Patterns

Rectangular Loop

A simple decrement-until-zero loop. CL and H0 point at the counter.

/−−−−−\ IP enters going E at top-left / · · Body: decrement counter (−) \−−−−−% % = exit S when grid[CL]=0, else E→N (loop back)

Conditional Branch (If-Then-Else)

Two paths merge back together. CL points at condition.

/ · · · · \ THEN path (grid[CL] ≠ 0) → · % · · · · & · → ELSE path (grid[CL] = 0) % = / if ≠0 (deflects E→N into THEN) & = \ if ≠0 (deflects S→E to merge)

Reversible Loop with Flag (Enter from Outside)

Use a flag cell to disambiguate first-entry from loop-back. CL switches between flag and counter.

/ · · \ loop-back redirect ! ! = \ if grid[CL]=0 (entry deflects, loop-back NOP) + increment flag (H0→flag) E > move H0 and CL to counter − W decrement counter, move H0 back \ · · % % = / if ≠0 (loop back N), else exit E < move CL back to flag on loop-back

Flag disambiguates: on entry flag=0, on loop-back flag≠0.

Breadcrumb Trail (Merge-Point Disambiguation)

For instruction-level merges (e.g., compiling jump targets), use GP to leave breadcrumbs.

Predecessor A corridor: P ] (write 1, advance GP) Predecessor B corridor: P P ] (write 2, advance GP) Reverse from merge body: [ Q (retreat GP, decrement — if now 0, was pred A) ...conditional routing back to correct predecessor...
