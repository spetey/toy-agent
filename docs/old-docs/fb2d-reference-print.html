fb2d Reference Sheet

2D reversible programming language · toroidal grid · von Neumann architecture · v0.5

Execution Model

The instruction pointer (IP) moves across a toroidal 2D grid. Every instruction is reversible — the simulator can step backward without history.

  1. Read the opcode at grid[IP_row, IP_col]
  2. Execute the instruction (mirrors may redirect IP direction)
  3. Advance IP one cell in its current direction (post-redirect)

Default grid: 8 × 16 (128 cells). All coordinates wrap toroidally. Values are bytes (0–255, mod 256 arithmetic).

Machine State

grid[r×c] program + data (byte array)
IP (row,col) instruction pointer position
DIR IP direction: N / E / S / W
CL control locus (flat index) — condition register
H0 data head 0 (flat index) — primary operand
H1 data head 1 (flat index) — secondary operand

All three heads (CL, H0, H1) are flat indices into the grid. Initial state: all at position 0, IP at (0,0) heading East.

Instruction Set — 26 Opcodes + NOP

NOP

Byte 0 (displayed as ·) and any undefined byte (27–255): IP passes through, no effect. Any byte sequence is a valid program.

Reversibility ↺ history-free

Every instruction has a unique inverse. Backward execution requires no history stack.

Simulator Commands

Program Loading

tape <code>Load code linearly onto grid (resets all state)
row <r> [c] <code>Place code along row r starting at col c (default 0)
col <c> [r] <code>Place code down column c starting at row r (default 0)
data <r> <c> <v…>Set raw values at (r,c)+; values can be numbers or opcode chars
cell <r> <c> [v]Get/set a single cell

Register Control

ip <r> <c>Set IP position
dir <N|E|S|W>Set IP direction
cl [r c | flat]Set/show CL register
h0 [r c | flat]Set/show H0 register
h1 [r c | flat]Set/show H1 register

Execution

step / s [n]Step forward n instructions (default 1)
back / b / r [n]Step backward n instructions (default 1)
run [n]Run forward n steps (default 100)
runback [n]Run backward n steps (default 100)

Display & State

showDisplay grid (opcode mnemonics + pointer overlays)
valsDisplay raw byte values
bothDisplay grid + values
traceToggle per-step execution trace
colorToggle ANSI color output

File & Session

save <file>Save full state to .fb2d file
load <file>Load state from .fb2d file
size <r> <c>Resize grid (resets state)
resetReset all state to initial
example <name>Built-in: bounce, loop, mirrors, branch, multiply
quit / q / exitExit simulator

Design Principles

Valid everywhereAny byte sequence is a valid program (undefined bytes are NOPs)
Fully reversibleEvery instruction has a unique inverse; backward execution needs no history
Von NeumannCode and data share the same grid — self-modifying programs possible
Toroidal topologyAll positions wrap — the grid has no edges
2D spatial flowControl flow via mirror reflections; loops are literal rectangles in space
