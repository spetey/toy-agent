# fb2d Reference Sheet

2D reversible programming language · toroidal grid · von Neumann
architecture · v0.5

## Execution Model

::: section
The instruction pointer (IP) moves across a toroidal 2D grid. Every
instruction is reversible --- the simulator can step backward without
history.

1.  **Read** the opcode at `grid[IP_row, IP_col]`
2.  **Execute** the instruction (mirrors may redirect IP direction)
3.  **Advance** IP one cell in its current direction (post-redirect)

Default grid: 8 × 16 (128 cells). All coordinates wrap toroidally.
Values are bytes (0--255, mod 256 arithmetic).
:::

## Instruction Set --- 26 Opcodes + NOP

:::: section
[Mirrors · Control Flow]{.cat-tag .tag-mirror}

  Char             Byte   Instruction                                        Inverse
  ---------------- ------ -------------------------------------------------- --------------
  [/]{.op-char}    1      Unconditional `/` reflect                          self-inverse
  [\\]{.op-char}   2      Unconditional `\` reflect                          self-inverse
  [%]{.op-char}    3      `/` reflect if `grid[CL] ≠ 0`, else pass through   self-inverse
  [?]{.op-char}    4      `/` reflect if `grid[CL] = 0`, else pass through   self-inverse
  [&]{.op-char}    5      `\` reflect if `grid[CL] ≠ 0`, else pass through   self-inverse
  [!]{.op-char}    6      `\` reflect if `grid[CL] = 0`, else pass through   self-inverse

::: mirror-diagram
**/**  reflect:  E [↔]{.arr} N    S [↔]{.arr} W     [(dr,dc) →
(−dc,−dr)]{style="color:var(--text-dim)"}\
**\\**  reflect:  E [↔]{.arr} S    N [↔]{.arr} W     [(dr,dc) →
(dc,dr)]{style="color:var(--text-dim)"}
:::

Conditional mirrors act as branching: the IP either reflects (taking the
turn) or passes straight through, based on grid\[CL\].

::: section
[Data Operations]{.cat-tag .tag-data}

  Char            Byte   Instruction                                                                                      Inverse
  --------------- ------ ------------------------------------------------------------------------------------------------ --------------
  [+]{.op-char}   15     `grid[H0]++`  (mod 256)                                                                          −
  [−]{.op-char}   16     `grid[H0]--`  (mod 256)                                                                          \+
  [.]{.op-char}   17     `grid[H0] += grid[H1]`  (accumulate add)                                                         ,
  [,]{.op-char}   18     `grid[H0] -= grid[H1]`  (accumulate sub)                                                         .
  [x]{.op-char}   19     `swap(grid[H0], grid[H1])`                                                                       self-inverse
  [F]{.op-char}   20     If `grid[CL] ≠ 0`: `swap(grid[H0], grid[H1])`  [(Fredkin gate)]{style="color:var(--text-dim)"}   self-inverse
:::

::: section
[CL Data · Bridging]{.cat-tag .tag-bridge}

  Char            Byte   Instruction                                                                                                  Inverse
  --------------- ------ ------------------------------------------------------------------------------------------------------------ --------------
  [G]{.op-char}   21     `swap(CL_register, grid[H0])`  [(indirect CL --- load/store CL as value)]{style="color:var(--text-dim)"}     self-inverse
  [T]{.op-char}   22     `swap(grid[CL], grid[H0])`  [(bridge --- transfer between CL and H0 cells)]{style="color:var(--text-dim)"}   self-inverse
:::

::: section
[NOP]{.cat-tag style="background:#8b949e20; color:var(--text-dim)"}

Byte `0` (displayed as `·`) and any undefined byte value (27--255) are
NOPs --- the IP passes through with no effect. This makes the language
**\"valid everywhere\"**: any byte sequence is a valid program.
:::

## Reversibility [↺ history-free]{.reversible-badge}

::: section
Every instruction has a well-defined inverse. The simulator reverses
execution *without* storing a history stack --- it deduces the prior
state from the current state alone:

  Forward                  Backward           Notes
  ------------------------ ------------------ --------------------------------
  `+ / −`                  `− / +`            Arithmetic inverses
  `. / ,`                  `, / .`            Accumulate inverses
  `N / S`, `E / W`         `S / N`, `W / E`   Direction reversal (all heads)
  `/ \ x F G T`            themselves         Self-inverse (involutions)
  Conditionals `% ? & !`   themselves         Same condition re-evaluated

Reverse step: back up IP one cell in the opposite direction, read that
cell\'s opcode, re-evaluate any condition, apply the inverse operation.
:::

## Simulator Interface Commands

::: section
### Program Loading

  --------------------------- -----------------------------------------------------------------
  tape \<code\>               Load code linearly onto grid (resets all state)
  row \<r\> \[c\] \<code\>    Place code along row *r* starting at col *c* (default 0)
  col \<c\> \[r\] \<code\>    Place code down column *c* starting at row *r* (default 0)
  data \<r\> \<c\> \<v...\>   Set raw values at (r,c)+; values can be numbers or opcode chars
  cell \<r\> \<c\> \[v\]      Get/set a single cell
  --------------------------- -----------------------------------------------------------------
:::

::: section
### Register Control

  -------------------- ----------------------
  ip \<r\> \<c\>       Set IP position
  dir \<N\|E\|S\|W\>   Set IP direction
  cl \[r c \| flat\]   Set/show CL register
  h0 \[r c \| flat\]   Set/show H0 register
  h1 \[r c \| flat\]   Set/show H1 register
  -------------------- ----------------------
:::

::: section
### Execution

  -------------------- --------------------------------------------
  step / s \[n\]       Step forward *n* instructions (default 1)
  back / b / r \[n\]   Step backward *n* instructions (default 1)
  run \[n\]            Run forward *n* steps (default 100)
  runback \[n\]        Run backward *n* steps (default 100)
  -------------------- --------------------------------------------
:::

::: section
### Display & State

  ------- ----------------------------------------------------
  show    Display grid (opcode mnemonics + pointer overlays)
  vals    Display raw byte values
  both    Display grid + values side by side
  trace   Toggle per-step execution trace
  color   Toggle ANSI color output
  ------- ----------------------------------------------------
:::

::: section
### File & Session

  ------------------ --------------------------------------------------------------------------
  save \<file\>      Save full state to `.fb2d` file
  load \<file\>      Load state from `.fb2d` file
  size \<r\> \<c\>   Resize grid (resets state)
  reset              Reset all state to initial
  example \<name\>   Load built-in example: `bounce`, `loop`, `mirrors`, `branch`, `multiply`
  quit / q / exit    Exit simulator
  ------------------ --------------------------------------------------------------------------
:::

## Design Principles

::: section
  ------------------- --------------------------------------------------------------------------------------
  Valid everywhere    Any byte sequence is a valid program (undefined bytes are NOPs)
  Fully reversible    Every instruction has a unique inverse; backward execution requires no history stack
  Von Neumann         Code and data share the same grid --- self-modifying programs are possible
  Toroidal topology   All positions wrap --- the grid has no edges
  2D spatial flow     Control flow via mirror reflections; loops are literal rectangles in space
  ------------------- --------------------------------------------------------------------------------------
:::
